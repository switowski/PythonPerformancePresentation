How to benchmark ?

Start REPL: 

$ ipython # Using Python 2.7.6 version

Write a function: 

def test():
    arr = []
    for i in range(100):
    	arr.append(i)

timeit: 

%timeit test

Et voilà: 

100000000 loops, best of 3: 7.15 ns per loop

So I'm gonna show you different ways to code some common programming structures along with the benchmarks. Benchmark numbers are not important. What is important is how easily you can get some speed improvements. $$ So I'm gonna use IPython and it's magic function called %timeit, which is just wrapper around Python's timeit library. If you are not using IPython - I strongly recommend it, it's way more powerfull than the standard Python REPL with autocompletion, some magic functions, syntax highlighting and other candies.'

Counting the number of elements in list

def slow_list_count(array):
how_many = 0
for element in array:
    how_many += 1
return how_many
def faster_list_count(array):
    return len(array)

Difference ?

>>> ARR=range(1,1000000) # 1 million, because big data
>>> %timeit slow_list_count(ARR)
10 loops, best of 3: 26.5 ms per loop
>>> %timeit fast_list_count(ARR)
10000000 loops, best of 3: 96.7 ns per loop # ~274 000 times faster
Let's start with something simple. Let's say you want to count the number of elements in a list. $$ You can easily write you own function, that contains just a for loop that increments a variable. There is nothing wrong with this code, except that it's slow. $$ You can achieve the same result using the built-in len() function. $$ Any difference ? $$ Yes, the second function is two hundred seventy thounsands times faster. So, try to use the built-in function when you can. Python is not Ruby, there is just a handful of built-in functions, so it's not so hard to remember them.
